# Code Style -- html

## 规则

1. 使用css不使用style
2. 在布局时能不使用具体的数字就不使用
3. 使用flex进行布局
4. 单位使用rem
5. 功能完成，不是在自己电脑上能运行，是要整个项目能正常运行部署
6. 把问题拿出，不要把它遗忘在开发的过程中，在代码中加入 todo 说明，添加到github的issues
7. 先思考后写代码，从命名开始
8. 在对外提供的接口中，统一错误编号及提示
9. 如果要使用使用全局变量，给出足够的理由，因为它很难测试
10. 提交代码的要求， 说明 格式化 编译通过，如果提交编译不通过的代码需要有特别的理由

## Name 

1. 所有源代码文件名，使用小写，加下划线
2. 所有目录文件名，使用小写，加下划线
3. 命名使用有明确函义的英文单词
